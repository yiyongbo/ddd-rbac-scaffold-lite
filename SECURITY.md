# 安全策略

## 支持版本

当前项目仍处于早期开发阶段，暂时仅维护主分支代码。

| 版本 / 分支 | 是否支持安全更新 |
| ---------- | ---------------- |
| main       | ✅ 支持           |

后续如果项目发布正式版本，将会在此处补充具体的版本支持策略。

## 安全漏洞报告

如果你发现本项目存在潜在安全漏洞，请不要直接创建公开 Issue 暴露漏洞细节。

请优先通过 GitHub 的 Private vulnerability reporting 功能向维护者私下报告。如果当前平台不支持私密漏洞报告，也可以通过项目维护者公开提供的联系方式进行反馈。

报告安全漏洞时，建议尽量包含以下信息：

- 漏洞的简要描述
- 影响范围或受影响模块
- 复现步骤
- 可能造成的安全影响
- 相关日志、截图或示例代码
- 你认为可行的修复建议

维护者会在收到报告后尽快进行确认、评估和修复。

## 安全问题范围

以下类型的问题适合通过安全漏洞报告渠道反馈：

- 认证绕过问题
- 授权或权限校验缺陷
- 越权访问问题
- Token、JWT 或会话管理问题
- 密码存储、加密或校验问题
- 敏感信息泄露
- SQL 注入或其他注入风险
- 不安全的反序列化问题
- 依赖组件中的已知安全漏洞
- 其他可能影响系统安全性的问题

## 非安全问题

以下问题通常不属于安全漏洞范围，可以通过普通 Issue 反馈：

- 功能建议
- 文档错误
- 代码风格问题
- 普通 Bug
- 构建失败
- 使用疑问
- 兼容性问题

## 处理原则

在漏洞被确认并修复前，请不要公开传播漏洞细节、复现方式或利用代码。

维护者会根据漏洞影响范围和严重程度，尽量及时完成修复，并在必要时发布修复说明。
